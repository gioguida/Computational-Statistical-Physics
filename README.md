# Particle Track Reconstruction as a Spin-Glass Optimization Problem

## Overview

Track reconstruction in high-energy physics (e.g. at CERN) is a large combinatorial problem. This project maps a simplified 2D version of the task onto an Ising-like spin-glass model. Potential track segments between detector layers are treated as binary variables (spins). By designing a Hamiltonian that rewards smooth trajectories and strongly penalizes bifurcations, track finding becomes an energy minimization problem solved via simulated annealing.

## How the Simulation Works

The pipeline runs in four sequential stages, all controlled from `scripts/config.yaml`:

### 1. Data Generation (Python)
- Generates random 2D particle trajectories (circular arcs) crossing 5 concentric detector layers at radii [1, 2, 3, 4, 5].
- Each trajectory is parameterised by a random angle ПҶ, radius r вҲҲ [4, 20], and curvature sign k вҲҲ {вҲ’1, +1}.
- Outputs `data/ground_truth_hits.csv` (clean) and `data/training_hits.csv` (with optional Gaussian smearing and Poisson fake hits).

### 2. Interaction Matrix Construction (C++)
- Reads hits from the CSV and groups them by detector layer.
- Creates all possible segments connecting hits on consecutive layers.
- Builds a sparse Ising coupling matrix **J** encoding three interaction types:
  - **Reward** (J > 0): two segments that form a smooth curve (dot-product alignment within Оё_max вүҲ 30В°).
  - **Merge penalty** (J = вҲ’10): two segments ending at the same hit (a track cannot split).
  - **Fork penalty** (J = вҲ’10): two segments starting from the same hit.
- Outputs `segments.csv`, `J_edges.csv`, and `interaction_meta.json` inside the run folder.

### 3. Simulated Annealing (C++)
- Initialises an Ising model where each spin s_i вҲҲ {вҲ’1, +1} represents whether segment i is selected.
- Uses the Metropolis-Hastings algorithm with the Hamiltonian:

  ```
  H = вҲ’вҲ‘_ij J_ij s_i s_j вҲ’ вҲ‘_i h_i s_i
  ```

- Temperature is cooled from T_max = 5.0 down to T_min = 0.05 in steps of 0.05, with 50 equilibration sweeps per temperature.
- Stops early if the energy converges (tolerance = 0.001).
- Outputs `final_state.csv` and `annealing_meta.json`.

### 4. Visualisation / Evaluation (planned)
- Intended to compare reconstructed tracks against ground truth and visualise results.

## Configuration

All parameters are set in `scripts/config.yaml`:

| Section | Key | Default | Description |
|---|---|---|---|
| `switches` | `build` | `true` | Recompile C++ binaries |
| `switches` | `generate` | `true` | Re-generate hit data |
| `switches` | `interaction` | `true` | Run interaction stage |
| `switches` | `anneal` | `true` | Run annealing stage |
| `interaction` | `theta_max` | `0.5236` (ПҖ/6) | Max angle between aligned segments (rad) |
| `interaction` | `penalty` | `10.0` | Coupling penalty for forks/merges |
| `annealing` | `t_max` | `5.0` | Initial temperature |
| `annealing` | `t_min` | `0.05` | Final temperature |
| `annealing` | `t_step` | `0.05` | Cooling step |
| `annealing` | `eq_sweeps` | `50` | Metropolis sweeps per temperature |
| `annealing` | `seed` | `42` | RNG seed |

To reuse the hits/segments from a previous run and only re-run annealing, set `run.run_id` to an existing run folder name and disable the earlier stages.

## Prerequisites

- **Python** >= 3.12
- **CMake** >= 3.15
- A C++17-compatible compiler (GCC, Clang, MSVC)

## Running the Simulation

### Option A вҖ” with `uv` (recommended)

[`uv`](https://docs.astral.sh/uv/) manages the virtual environment and dependencies automatically.

```bash
# Install uv if not already installed
pip install uv

# Install dependencies and run
uv run python scripts/control_panel.py
```

That's it. `uv` reads `pyproject.toml`, creates a `.venv` if needed, and launches the full pipeline.

### Option B вҖ” with standard `pip`

```bash
# Create and activate a virtual environment
python -m venv .venv

# On Linux/macOS:
source .venv/bin/activate
# On Windows:
.venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run the pipeline
python scripts/control_panel.py
```

> **Note:** Both options invoke CMake internally (if `switches.build: true`) to compile the C++ binaries before running the Python stages. Make sure `cmake` is available on your PATH.
